# Contributing

## Scope

This repository is a public technical face for DABBAH 666. Contributions should strengthen runtime law, structural coherence, evaluation depth, and public technical clarity.

## Contribution standard

Please keep changes aligned with these rules:

- do not add decorative or purely cosmetic changes
- do not add dead code or redundant wrapper layers
- do not weaken proof, route, truth, verdict, lineage, or runtime-mode clarity
- do not merge unrelated refactors with substantive logic changes
- keep evaluation artifacts reproducible
- prefer small, structurally justified changes

## Recommended process

1. Open an issue describing the structural reason for the change.
2. Keep pull requests focused and minimal.
3. Include or update tests when behavior changes.
4. Explain how the change affects runtime law, governance, or constitutional evaluation.

## Review posture

Changes should be judged by structural truth, test evidence, and architectural coherence rather than by cosmetic tidiness alone.
